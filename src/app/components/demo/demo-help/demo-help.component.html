Given a planning task like delivering packages or scheduling a Mars-Rover mission the goal is to find a plan. A plan is a sequence of actions which describes what you have to do to solve the planning task. If you habe to deliver packages the plan tells you for example where you have to drive and where you have to load and unload the packages.

The number of actions you can perform are limited, for example your truck has a limited amount of fuel and the Mars-Rover has a battery.

Additionally not every plan is equally good in your eyes. Maybe you prefer plans where: "The route over downtown is not used, because there is always to much traffic." or "Important tasks are scheduled for the rover before not so important once as the battery may die early."

The goal is to find the best plan possible given the resource constraints and your preferences.

To find the best plan we use Iterative Planning with human guidance. This approach is explained in the next step.

Iterative Planning with human guidance works the following way. You define a set of properties you want to be satisfied by the plan (in the following called hard-goals). The planner then computed a plan which satisfies these hard-goals if possible. If there is no plan you have to remove properties from the hard goals until there exists a plan.

If the planner finds a plan two cases are possible:

1. The plan satisfies all the properties you want and does not satisfy any properties you do not want. In this case you are ready. 2. The plan does not satisfy all the properties you would like to.

In case 2. you may simply want to enforce the missing properties. But what happens if this is not possible? It might be the case that there exists no such plan, because you have for example not enough fuel or the properties you would like to be satisfied somehow contradict each other.

Instead of randomly trying to find out, for which property selection there exists a plan, you can ask questions about properties not satisfied in the current plan. Such a question could for example be: "Why don't you use the less busy route around down town?".

The system while then provide an answer describing which currently satisfied properties can no longer be satisfied if you would satisfy the property/ies in your question. "If you use the route around down town, then you can not longer deliver package 1", could for example be the answer of the sample question.
